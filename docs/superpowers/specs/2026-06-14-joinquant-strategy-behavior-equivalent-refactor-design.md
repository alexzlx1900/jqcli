# 聚宽首板一进二策略行为等价重构设计

## 目标

基于聚宽策略 `algorithmId=b69c28a4f69ad63165772d3eddbdccd7` 创建一个全新的策略：

`首板一进二-混合版-行为等价重构-v1`

新策略必须严格保留原策略的交易行为，只改善代码组织、命名、注释和人类可读日志。原策略不得被修改。

## 行为等价契约

在相同回测参数下，新旧策略必须产生一致的：

- 调度时间与函数执行顺序
- 每日候选池与最终通过名单
- 买卖标的、下单时间和下单方向
- 目标金额、目标股数与订单样式
- 成交结果、持仓变化和交易结果记录
- 每日收益曲线与核心回测指标
- 结构化研究记录、候选决策记录和导出数据

允许变化的内容仅包括：

- 函数与局部变量命名
- 代码分区、函数排序和无副作用的辅助函数提取
- 注释、文档字符串和格式
- 面向人工阅读的日志文本

任一交易行为、结构化记录或核心指标发生变化，均视为重构失败。

## 明确禁止的变更

- 不修改阈值、公式、条件优先级或计算顺序。
- 不修改 `run_daily` 调度时间和注册顺序。
- 不修改回测与实盘的下单方式、价格样式或订单处理。
- 不修改异常发生后的默认值、跳过逻辑和继续执行方式。
- 不启用当前关闭的 ML 门控、市场承接门控或影子卖出评估。
- 不修复已识别的策略逻辑问题，包括回测 09:26 按开盘价成交的问题。
- 不因为清理代码而删除研究字段、运行时状态或兼容性分支。

## 重构方案

聚宽部署继续使用单一 Python 文件。文件内部按职责重新排序，并使用清晰的章节标题：

1. 策略元信息与参数
2. 初始化、运行时状态与每日调度
3. 盘前候选池构建
4. 集合竞价条件与候选评分
5. 仓位预算与买入执行
6. 持仓检查与卖出执行
7. 实盘订单适配与状态对账
8. 市场、主题、ML 与研究因子
9. 结构化记录、数据导出与基础工具

本次以低风险机械重构为主：

- 将重复的日志拼装提取为无副作用日志辅助函数。
- 将超长函数按现有阶段边界拆分，但保持调用与求值顺序。
- 为 A-F 条件、评分、仓位和风控流程补充简洁说明。
- 统一命名风格，但保留聚宽要求的生命周期入口函数名称。
- 保留所有已有全局状态字段和记录字段。

## 日志设计

日志采用统一前缀和稳定字段顺序，重点回答“发生了什么、为什么、影响多少资金”。

### 阶段摘要

```text
[盘前摘要] 初始=3048 强势=89 风险过滤后=62 市场=normal 回撤系数=1.00
[竞价摘要] 目标=62 条件通过=3 最终通过=1 拒绝=2 可用预算=1,940,000
[收盘摘要] 持仓=0 当日买入=0 当日卖出=1 总资产=3,571,000
```

### 决策日志

```text
[买入决策] 603598.XSHG 条件=E 评分=0.76 仓位系数=0.76 预算系数=1.00
[买入提交] 603598.XSHG 模式=回测 参考价=8.42 目标金额=1,474,400
[卖出决策] 603598.XSHG 原因=未涨停止盈 收益=81.01%
```

### 跳过与异常日志

```text
[候选拒绝] 000001.XSHE 阶段=竞价条件 原因=竞昨比不足
[数据异常] 模块=市场观察 动作=保留默认值 交易继续=true 错误=<message>
```

日志辅助函数只负责格式化和调用 `log.info`，不得修改状态或重新计算交易因子。日志中需要展示的数值必须复用已经计算出的局部变量或记录字段。

## 新策略创建与隔离

1. 从原策略重新获取最新源码并保存为本地工作副本。
2. 使用原源码创建新策略，确认新策略 ID 与原策略 ID 不同。
3. 所有后续编辑只作用于新策略。
4. 每次上传前后重新读取策略元数据，防止误修改原策略。
5. 原策略只用于读取源码、读取基线回测和进行结果对比。

## 验证设计

### 静态验证

- Python 语法编译通过。
- 调度注册、策略参数字典和关键订单调用与原策略一致。
- 对关键函数建立 AST/文本级契约检查，防止阈值、调度时间和订单样式漂移。
- 原策略远端元数据与源码摘要在任务前后保持一致。

### 快速验证

- 新策略执行聚宽 compile 回测。
- 如失败，先读取错误日志，再修正结构性问题。
- 使用短区间回测比对买卖行为和日志可读性。

### 正式行为等价验证

统一参数：

```text
开始日期：2026-01-01
结束日期：2026-06-13
初始资金：2,000,000
频率：day
```

对比项目：

- 完整订单序列
- 完整交易序列
- 每日收益、持仓和资金曲线
- 总收益、最大回撤、胜率、交易次数和其他核心指标
- 结构化候选与交易结果记录

订单与交易记录要求逐字段一致；收益曲线和指标按聚宽返回的序列化精度一致。日志文本不参与等价比较，但日志覆盖的关键阶段和拒绝原因必须完整。

## 失败处理

- 编译失败：读取错误日志，只修复结构或引用问题。
- 行为不一致：定位首个发生差异的日期和阶段，回退对应重构单元后重新验证。
- 原策略发生变化：立即停止写操作，重新确认目标策略 ID。
- 聚宽返回数据不完整：保留已有证据并重试，不用部分结果宣称等价。

## 完成标准

- 新策略已创建，原策略未被修改。
- 新策略代码职责分区清晰，关键长函数得到有边界的整理。
- 日志能够直观呈现盘前筛选、竞价决策、买卖提交、跳过原因和收盘摘要。
- compile 回测通过。
- 正式回测的订单、交易、收益曲线和核心指标与原策略严格一致。
- 输出新策略 ID、验证结果和仍然保留的原始策略风险说明。
